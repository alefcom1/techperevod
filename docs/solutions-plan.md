# Решения по 4 измерениям + PDF-примеры переводов — план

Дата: июль 2026. Две связанные задачи: (1) реорганизация навигации и разделов
по образцу RushTranslate — группировка «решений» по языку / документу /
отрасли / сценарию использования; (2) блок «Примеры перевода» с PDF-образцами
на отраслевых и языковых страницах.

---

## Задача 2. Блок «Примеры перевода» с PDF

### Что делаем

На каждой отраслевой и языковой странице — блок из 4 карточек PDF-примеров
(«документ этой отрасли → на 4 разных языка» / «документ на языке страницы»).
Языки распределяем по страницам детерминированно-разнообразно (seed от slug),
из пула: en, de, fr, es, it, pl, cs, nl, tr, pt, fi, sv.

### PDF-образцы — генерация

- **Статически, скриптом в репо** (`scripts/generate-samples.mjs`,
  запуск вручную при изменении данных, вывод в `public/samples/*.pdf`) —
  никакой runtime-генерации, ноль нагрузки на прод.
- Библиотека: `pdfkit` (Node, dev-dependency) + TTF-шрифт с кириллицей
  (DejaVu Sans из системы или вкоммиченный в repo `assets/fonts`).
- Контент образцов: `src/data/sampleDocs.ts` — для каждого документа-типа
  фрагмент оригинала (ru) + переводы на языки пула (готовим переводы
  агентами, проверяю выборочно я/носители).
- Каждый PDF:
  1. **Стр. 1** — «перевод»: шапка бюро (Техперевод.com + контакты),
     заголовок типа документа, двуязычная таблица или текст перевода,
     диагональный водяной знак «ПРИМЕР / SAMPLE» (полупрозрачный, поверх);
  2. **Стр. 2** — сертификат точности перевода **на языке перевода**
     (по образцу RushTranslate): заявление о точности, описание документа
     и языковой пары, ФИО/подпись представителя бюро, фирменный бланк;
     водяной знак «ПРИМЕР» тоже ставим (это образец, не боевой сертификат).
- Имена файлов: `public/samples/{docType}-{src}-{tgt}.pdf`
  (напр. `pasport-bezopasnosti-ru-de.pdf`).

### Блок на страницах

- Компонент `SampleTranslationsSection` (карточки: тип документа, пара
  языков, превью-иконка, ссылка на PDF target=_blank).
- Данные соответствия: в `sampleDocs.ts` — маппинг отрасль→типы документов,
  язык→типы документов; выбор 4 штук — детерминированный (hash slug).
- JSON-LD не нужен; alt/aria — обязательно.

### Этапы

1. Каркас: pdfkit-скрипт + 1 пилотный PDF (SDS ru→de) — согласовать вид.
2. Контент образцов для 6–8 типов документов × 4–6 языков — волна агентов
   (перевод фрагментов), моя проверка.
3. Генерация всех PDF, компонент блока, врезка на otrasli/perevod страницы.
4. Деплой + переобход в Вебмастере.

### Правовая заметка

Все образцы — вымышленные данные (без реальных имён/компаний/номеров),
водяной знак «ПРИМЕР» на каждой странице, сертификат — от имени нашего же
бюро (не имитация третьих лиц). Это витрина качества, аналог примеров на
/primery-perevodov, но в «боевом» оформлении PDF.
