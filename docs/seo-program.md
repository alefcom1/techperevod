# techperevod.com — инструкции по запуску и программа SEO-генерации

Дата: июль 2026. Документ состоит из двух частей: **(1)** что нужно сделать владельцу
руками прямо сейчас, **(2)** полная программа генерации SEO-страниц с готовыми
ТЗ-промптами для агентов.

## Часть 1. Что сделать сейчас (по пунктам)

### 1.1. Задеплоить текущую версию на сервер

```bash
cd ~/techperevod
git fetch origin && git reset --hard origin/main
npm install
npm run build
pm2 restart techperevod --update-env
```

Проверить после деплоя:
- главная: чистый hero, переводчик — второй блок, работает автоперевод при паузе в наборе;
- https://techperevod.com/uslugi/perevod-kitajskogo-oborudovaniya открывается, в футере есть колонка «Услуги»;
- https://techperevod.com/admin — вход по паролю из `ADMIN_PASSWORD`, во вкладке
  «Тексты главной» появилась карточка «Блок переводчика».

### 1.2. Завести цели в Яндекс.Метрике

Метрика (счётчик 110336524) → Настройки → Цели → «Добавить цель» → тип **JavaScript-событие**.
Создать 6 целей с точно такими идентификаторами:

| Идентификатор | Что означает |
|---|---|
| `translate_done` | Успешный перевод в бесплатном переводчике |
| `translate_limit` | Пользователь упёрся в дневной лимит — **горячий лид** |
| `translate_like` | Оценил перевод 👍 |
| `translate_dislike` | Оценил перевод 👎 |
| `doc_cta` | Клик «Перевести документ» или карточка «перетащите документ» |
| `swap_langs` | Поменял направление перевода |

После этого: Отчёты → Конверсии. Полезный сегмент для ретаргетинга: посетители
с ≥2 достижениями `translate_done`.

### 1.3. Подключить Яндекс.Вебмастер и Google Search Console

Без этого новые страницы будут индексироваться неделями.

1. webmaster.yandex.ru → добавить сайт techperevod.com → подтвердить права (DNS-запись через reg.ru DNSAdmin — самый простой способ).
2. Индексирование → «Файлы Sitemap» → добавить `https://techperevod.com/sitemap.xml`.
3. Индексирование → «Переобход страниц» → вручную отправить 7 новых URL `/uslugi/*`.
4. То же в Google Search Console (search.google.com/search-console): подтверждение через DNS, отправка sitemap.
5. Раз в неделю смотреть: Вебмастер → «Страницы в поиске» (что проиндексировано), «Запросы» (по каким фразам показываемся).

### 1.4. Контроль расходов на API перевода

Автоперевод при паузе в наборе увеличивает число запросов к Anthropic API
(лимит — 30 запросов/день с одного IP). Что сделать:

- следить за расходами в console.anthropic.com → Usage;
- для бесплатного переводчика можно переключиться на более дешёвую модель:
  в `.env.local` на сервере добавить `TRANSLATE_MODEL=claude-haiku-4-5-20251001`
  (в разы дешевле, для коротких фрагментов качество достаточное), затем
  `pm2 restart techperevod --update-env`.

### 1.5. Безопасность

- `ADMIN_PASSWORD` — длинный и уникальный (не из других сервисов);
- бессрочный GitHub-токен при желании можно в любой момент отозвать и выпустить новый (Settings → Developer settings → Personal access tokens);
- содержимое админки хранится на сервере в `~/techperevod/data/site-content.json` — этот файл не в git; при переезде на другой сервер скопировать вручную.

## Часть 2. Программа генерации SEO-страниц

### 2.0. Принципы (обязательны для каждого агента — вставлять в каждое ТЗ)

1. **Никакого тонкого контента.** Минимум 500 слов уникального текста на страницу,
   у флагманов 800+. Яндекс жёстко наказывает дорвеи: тексты пишутся для людей,
   с конкретикой (форматы, ГОСТы, сроки, примеры терминов), без «мы лучшие» и штампов.
2. **Уникальность.** Ни одного скопированного абзаца между страницами. Каждая
   страница отвечает на свой запрос своим содержанием.
3. **Данные отдельно от разметки.** Контент — в `src/data/*.ts` (по образцу
   `INDUSTRIES` и `SERVICES`), страницы — тонкие шаблоны с `generateStaticParams`.
4. **Каждая страница обязана иметь:** уникальный `<title>` ≤ 60 знаков,
   description 140–160 знаков, canonical, openGraph, JSON-LD `BreadcrumbList`
   (+ `FAQPage`, если есть FAQ), запись в `src/app/sitemap.ts`.
5. **FAQ 4–6 вопросов** — реальные вопросы клиентов, содержательные ответы по 2–4 предложения.
6. **Перелинковка hub-and-spoke:** каждая новая страница ссылается на 2–4 смежные
   (услуги ↔ отрасли ↔ языковые пары) и получает обратные ссылки из блоков
   «Смежные…» и футера.
7. **Дизайн-система:** только `tp-`-классы и `--tp-*`-токены, проверка в обеих темах.
   Переиспользовать компоненты: PageHero, SectionHeader, Card, FormatChip, IndustryCard,
   ScrollReveal, готовые body-шаблоны.
8. **Проверка агентом:** `npx tsc --noEmit` до чистого прогона. Агенту **нельзя**
   запускать `npm run build`/`dev`-сервер и git-команды — это делает оркестратор.

### 2.1. Что уже есть (инфраструктура)

- `src/data/site.ts` → `INDUSTRIES` (4 отрасли) → `/otrasli/[slug]` через `IndustryPageBody`;
- `src/data/services.ts` → `SERVICES` (7 услуг) → `/uslugi/[slug]` через `ServicePageBody` (FAQ-аккордеон + JSON-LD уже реализованы — переиспользовать);
- `src/app/sitemap.ts` — центральная карта сайта;
- футер-колонки в `FOOTER_COLUMNS` (`src/data/site.ts`);
- админ-панель автоматически подхватывает новые отрасли из `INDUSTRIES` (SEO-вкладка).

## Часть 3. Продуктовые этапы за пределами SEO (дорожная карта)

Эти этапы — не страницы, а функциональность. Порядок — по соотношению
«эффект / сложность». Каждый запускается отдельным решением после SEO-волн
или параллельно с ними.

### 3.1. Мгновенная оценка документа без менеджера (приоритет 1)

Загрузил DOCX/PDF → сервер считает слова и повторы → через минуту цена, срок
и кнопка «Оформить заказ». Главный дифференциатор против бюро («оставьте заявку,
менеджер посчитает завтра»). Технически: парсинг DOCX/PDF на сервере, подсчёт
слов/сегментов, простая формула цены из тарифов, письмо-уведомление владельцу.

### 3.2. Отчёт о качестве перевода

После сдачи заказа клиент видит: какие термины взяты из его термбазы, сколько
сегментов правил инженер (с diff-просмотром), какие QA-проверки пройдены (числа,
единицы, теги). Превращает «чёрный ящик» в прозрачный процесс — аргумент для B2B,
которого нет ни у кого на рынке. Начать с демо-версии в кабинете.

### 3.3. Умная маршрутизация на человека (модель Unbabel)

AI оценивает уверенность по каждому сегменту; редактор проверяет только рискованные
15–20 %. Продаётся как «редактура человеком в разы дешевле — проверяем только то,
где есть риск».

### 3.4. Telegram-бот с корпоративной термбазой

Быстрый перевод фрагментов прямо в рабочем чате, с терминологией компании
(привязка к аккаунту платформы). Западные конкуренты делают Slack-интеграции;
Telegram-first — естественное преимущество для российского рынка, аналогов нет.

### 3.5. Локализация сайтов «на лету» (модель Weglot/MotionPoint)

Отдельный продукт на перспективу: клиент вставляет JS-сниппет или подключает
поддомен (en.site.ru) — его сайт переводится прокси-слоем без переделки кода,
переводы редактируются в кабинете, термбаза общая с документами. На российском
рынке аналога нет вообще. Большая разработка (прокси-сервер, кэш переводов,
визуальный редактор) — запускать после того, как заработают 3.1–3.2, как второй
продукт компании с отдельной посадочной страницей и тарифами.
