# Сравнение с конкурентами: чего не хватает сайту для привлечения клиентов

Дата: июль 2026. Анализ, без реализации.

## С кем сравниваем

Две группы конкурентов, потому что техперевод.com играет на двух полях сразу:

1. **Российские бюро технических переводов** (b2bperevod.ru, academperevod.ru, simwell.ru, perevodof.ru, mskperevod.ru, buro-rtt.ru) — конкуренты за клиента «мне нужно перевести документацию».
2. **SaaS-платформы перевода/локализации** (Smartcat, Weglot, Lokalise, Crowdin) — конкуренты за клиента «мне нужен инструмент/API/виджет».

## Что у нас уже есть и не хуже конкурентов

- 40+ SEO-страниц (услуги, отрасли, языковые пары, блог, словарь) — глубже, чем у большинства российских бюро.
- **Мгновенная оценка файла с реальным парсингом** — у бюро-конкурентов это почти всегда «пришлите файл, менеджер посчитает». Наше преимущество, его надо громче продавать.
- Рабочий бесплатный переводчик на главной — ни у одного из шести бюро такого нет.
- Кабинет с реальными аккаунтами, отчёт о качестве, маршрутизация, публичный API, виджет локализации — по фичам это уже мини-Smartcat.
- Дизайн-система и вид «продукта», а не «бюро с сайтом 2012 года».

## Критические пробелы (по убыванию важности)

### 1. Заявки уходят в никуда — форма контактов фиктивная ⛔

`ContactsBody.tsx`: `onSubmit={() => setSent(true)}` — заявка **никуда не отправляется**. Ни письма, ни записи в БД, ни уведомления. То же с кнопкой «Заказать проверку» в кабинете. Это единственный настоящий блокер: весь SEO-трафик, который мы собрали волнами 0–5, упирается в форму-пустышку.

**Что делать:** эндпоинт `/api/leads` → запись в БД (таблица `Lead`) + мгновенное уведомление в Telegram через Bot API (обычный `fetch` к `api.telegram.org`, ноль зависимостей, бесплатно, работает с reg.ru) + просмотр заявок в существующей админке. SMTP (nodemailer) — вторым каналом, если нужен email. Прикреплённый файл из формы прогонять через уже готовый `/api/quote` — менеджер получает заявку сразу с посчитанной оценкой.

### 2. Нет телефона и мессенджеров

У всех шести бюро-конкурентов телефон в шапке — в российском B2B это до сих пор базовое ожидание. У нас только email и текстовый «@techperevod». Нужны: телефон в шапке/футере, кликабельные кнопки Telegram (`t.me/...`) и WhatsApp (`wa.me/...`). Реализация тривиальна, **но нужны реальные контакты от вас** (номер, актуальный TG).

### 3. Нет социальных доказательств

Стандартный набор конкурентов, которого у нас нет совсем:
- **Кейсы с цифрами** («перевели 400 страниц мануала станка за 6 дней, сэкономили 30% на повторах») — по данным исследований B2B-конверсии это сильнейший trust-сигнал. У нас есть готовый паттерн `posts.ts` → та же структура для `cases.ts` + `/keysy/[slug]`.
- **Примеры перевода** (фрагмент оригинал/перевод) — есть у perevodof, academperevod.
- **Логотипы клиентов и отзывы** — нужен реальный материал от вас; выдумывать нельзя.
- **Счётчики** («12 лет», «8 млн слов») — только если цифры настоящие.
- ISO 17100 — в отрасли это аргумент «+15–30% к чеку» (по данным TranslationCert), но сертификация — офлайн-процесс, на сайте лишь отразим, когда будет.

### 4. Нет цены «за страницу»

Все бюро публикуют якорь «от 450 ₽/стр» — именно по нему покупатель сравнивает предложения в выдаче. Наши тарифы — SaaS-подписки по словам. Для «бюро-половины» бизнеса нужна простая таблица на /tarify и страницах услуг: языковая пара → ₽/страница (250 слов). Данные уже есть (1,5 ₽/слово ≈ 375 ₽/стр — конкурентно!).

### 5. Бесплатный тестовый перевод как CTA

Частый крючок конкурентов — «тестовый перевод страницы бесплатно». У нас он **уже технически есть** (виджет 2000 знаков + мгновенная оценка) — надо лишь оформить как оффер: «Пришлите документ — страницу переведём бесплатно, оценку дадим за 2 минуты».

### 6. Нет юридической обвязки для B2B

- Footer-ссылка «Конфиденциальность и 152-ФЗ» ведёт на маркетинговую страницу /bezopasnost, а **настоящей политики обработки ПДн нет** — это требование 152-ФЗ при наличии форм и блокер модерации Яндекс.Директа.
- Нет реквизитов юрлица (ИНН/ОГРН), оферты, упоминания работы по счёту/ЭДО/НДС — первое, что спрашивает снабжение завода.
- Нужны страницы: `/politika-pd`, `/oferta`, реквизиты в футере. **Нужны данные юрлица от вас.**

### 7. Мелкие технические SEO-пробелы

- Нет og-image (соцсети/мессенджеры показывают ссылку без картинки — а ссылки на КП шлют именно в мессенджерах).
- Нет JSON-LD `Organization` на главной.
- Нет RSS для блога.

## Библиотеки (npm, совместимо с нашим стеком)

| Задача | Рекомендация | Комментарий |
|---|---|---|
| Уведомления о заявках | **Telegram Bot API через fetch** | Ноль зависимостей, мгновенно, бесплатно |
| Email-канал заявок | `nodemailer` | SMTP от почты reg.ru; западные Resend/Sendgrid — риск недоступности |
| PPTX/новые форматы в /api/quote | `jszip` + `fast-xml-parser` | Презентации часто просят переводить; SRT/VTT/HTML/MD парсятся вообще без библиотек |
| Антиспам форм | honeypot-поле + существующий in-memory лимит | Без сторонних капч; если прижмёт — Yandex SmartCaptcha |
| og-image | статичный PNG 1200×630 | Генератор не нужен, одна картинка на сайт + по одной на хабы |
| Валидация форм | `zod` (опционально) | Формы простые, можно и руками |

Из репозитория переиспользуем: паттерн `posts.ts`/`PostPageBody` → кейсы; `/api/quote` → авто-оценка вложений в заявках; админка → просмотр заявок; Метрика-цели (`reachGoal`) уже размечены — добавить цель `lead_submit`.

## Предлагаемый порядок работ

| # | Что | Размер | Нужно от вас |
|---|---|--------|--------------|
| 1 | Рабочие заявки: /api/leads + БД + Telegram-бот + админка | S | Создать бота (@BotFather), дать токен и chat_id |
| 2 | Телефон/мессенджеры в шапке и футере | XS | Номер телефона, ссылки TG/WA |
| 3 | Таблица цен за страницу + оффер «тестовый перевод бесплатно» | S | Подтвердить цены |
| 4 | Политика ПДн + оферта + реквизиты | S | Данные юрлица |
| 5 | Кейсы (3–5 шт., data-driven) + примеры перевода | M | Фактура реальных проектов (или согласуем обезличенные) |
| 6 | og-image, Organization JSON-LD, RSS | XS | — |
| 7 | PPTX/SRT/HTML в оценке файла | S | — |
| 8 | Отзывы/логотипы, лайв-чат, онлайн-оплата (ЮKassa), EN-версия сайта | L | Материал/решение позже |

**Рекомендую начать с №1 — это не «улучшение», а починка воронки: сейчас любой клиент, дошедший до формы, теряется молча.** Пункты 2–4 добиваются за один заход после того, как пришлёте контакты и реквизиты.
