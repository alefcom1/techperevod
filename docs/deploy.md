# Деплой сайта techperevod.com

Сервер: VPS под FASTPANEL, доступ по SSH (root). Сайт лежит в `~/techperevod`,
процесс управляется через PM2 под именем `techperevod`.

Ветка: всегда деплоим `main` с GitHub (`alefcom1/techperevod`) — никаких
других веток на сервере не используется.

## Команды деплоя (по порядку, на сервере)

```bash
cd ~/techperevod
git fetch origin && git reset --hard origin/main
npm install
npm run build
pm2 restart techperevod --update-env
```

## Важные нюансы

- `git reset --hard origin/main` затирает любые локальные правки на сервере
  в git-отслеживаемых файлах — поэтому реальные секреты (API-ключи,
  `ANTHROPIC_AUTH_TOKEN` и т.д.) должны лежать только в `.env.local` (он в
  `.gitignore`, деплой его не трогает). `.env` — закоммичен в репозиторий и
  переписывается каждым деплоем, туда секреты класть нельзя.
- Флаг `--update-env` у `pm2 restart` обязателен, если менялись переменные
  окружения (`.env.local`) — иначе PM2 продолжит работать со старыми
  значениями из памяти процесса.
- Контент админки (`data/site-content.json`) — не в git, при деплое не
  трогается, но и не переносится автоматически на другой сервер.

## Проверка после деплоя (минимум)

- главная открывается, работает переводчик;
- любая недавно добавленная страница из этой волны работы (например
  `/primery-perevodov` или `/standarty`) открывается и не даёт 404;
- `/admin` — вход по паролю из `ADMIN_PASSWORD`.

Это ровно тот процесс, который уже использовался раньше для деплоя сайта —
ничего нового изобретать не нужно, только повторить эти команды после
каждого `git push origin main` на GitHub.
